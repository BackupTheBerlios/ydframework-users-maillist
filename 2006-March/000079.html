Yes, the problem is in YDAjax (well.. is more a php4 problem to me). On 
php5 (my development version) everything went fine.

I had changed that because the form element is not really required. 
Sometimes we don't need a form.. but we want to use YDAjax. So we need a 
default value (null) for a form.
Another problem is when we have more than one YDForm. Currently we don't 
have a way to add YDForms to YDAjax.. we can just use one. So, we have 2 
problems here.

Your problem can be solved easily (because you are not using YDAjax 
without the form)...
But i'm planing to delete this parameter and YDAjax will have only the 
$template variable. Then, a new method will be added to YDAjax to add 
form elements. This way, we can have zero, one or more YDForms in 
YDAjax. This latest feature is planned but it's not trivial to implement 
because there are lots of form elements searches inside YDAjax.
