Hi maximus meridius

> I've spent the last couple of nights trying out YDajax. I'm quite 
> impressed as it integrates nicely with YDF. I thought it would be nice 
> to remove the dependency on YDform.

As you said in other email, since v2.4, YDForm object is not required 
anymore... but it's recommended ;)
 From that version it's possible to use YDAjax without the form logic 
_and_ it's possible to use up to 10 YDForms at the same time too :)


> One improvement would be to make use of the Behaviour library from 
> http://bennolan.com/behaviour/ . This allows the resulting HTML to be 
> quite clean since the resulting HTML does not have javascript event 
> attributes. This is BSD licensed.

I saw that but i think, in some complex cases, it can be ugly.
As YDAjax is created now, it works very well when you use YDForms 
(because you don't need to care about the js generated) and when you 
just add custom events (because you only need to care about the js 
function name, everything else is done by YDAjax).

> Another toy to look at for rich AJAX affects is Rico ( 
> http://openrico.org/rico/home.page ). This is apache licensed.

OpenRico seems very powerful. the only problem i see is that it's a pack 
of different features.. ajax, effects, drag&drop..
As you know, in YDF you have YDAjax (that is integrated with a effect 
lib too). ;)
