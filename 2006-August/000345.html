Hi

As i said in a latter email, it could be very usefull to have a standard 
way of representing results. YDError is an excellent choice for that but 
currently it can represent only results that are errors ( because normal 
results are returned without any modification ). This works but normal 
results have a user message associated.

I propose to:

- rename YDError to eg, YDResult

- add a new level ( OK ) to YDResult:

     @define( 'YD_ERROR_FATAL',   4 );
     @define( 'YD_ERROR_WARNING', 3 );
     @define( 'YD_ERROR_NOTICE',  2 );

     @define( 'YD_ERROR_OK',  1 );

     @define( 'YD_ERROR_NONE',    0 );

- add a new static method for the creation of normal results:

         function ok( $message, $value ){ ... }



What do you think?
