I was thinking more about this and the way reset() is working today is not
that good.

reset() should "reset" the object to it's initial state, or close to it. So
it should reset the fields, selects and also reset relations vars (not doing
this now).

I commited my changes to the svn repository and the main differences are:

- added resetRelations that reset all relations
- added parameter to reset( $all=false ) that resets everything in the
object including relations and other custom variables when true.
- modified resetValues to reset selects values too
