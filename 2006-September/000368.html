Hi David


> You can retrieve the SQL query that would be used without executing it 
> by setting the YDConfig variable YD_DBOBJECT_EXECUTE to false. Example:
> 
> YDConfig::set( 'YD_DBOBJECT_EXECUTE', false );
> 
> $obj->id = 1;
> $sql = $obj->delete();
> 
> YDConfig::set( 'YD_DBOBJECT_EXECUTE', true ); // get it back to normal use
> 
> Now $sql will have something like:
> 
> DELETE FROM table WHERE id = 1
> 
> Is this what you want?

Yes.. but it would be very useful to integrate them in the YDDatabaseObject.

Do you mind to add those? eg:


function deleteSQL(){

	$exe = YDConfig::get( 'YD_DBOBJECT_EXECUTE' );
	       YDConfig::set( 'YD_DBOBJECT_EXECUTE', false );
	$res = $this->delete();
	       YDConfig::set( 'YD_DBOBJECT_EXECUTE', $exe );
	return $res;
}
