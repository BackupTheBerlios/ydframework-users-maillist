Hi Graeme

> I guessed that the position field was for ordering children and can see 
> the benefits of doing so. A 'setChildPosition' method would be a nice 
> way to move a child element to a specific position. You could also have 
> moveChildToFirst( $id ), moveChildToLast ( $id ), moveChildLeft( $id, $n 
> ) and moveChildRight( $id, $n ). The last two would move the child N 
> positions (within the current parent).


.. but you would do that with YDDatabaseTree::moveNode, i think.

Currently, with moveNode( $id, $new_parent ) you can define the new 
parent only.. with a new moveNode, eg, moveNode( $id, $new_parent, 
$position ) then you have:

moveChildToFirst( $id )  == moveNode( $id, null, 1 )

moveChildToLast( $id )   == moveNode( $id, null, -1 )

but i don't know about moveChildLeft/moveChildRight..

A
+- B
|
+- C
|
+- D
|  +- X
|  +- Y
|
+--E

If moveChildLeft is applyed to X what would be the new parent? C? A?


My opinion, is that moveNode is the best way because it's the most basic 
and powerfull. With moveNode we define exactly what we want, eg:


moveNode( X, A, 1 );   or    moveNode( X, C, 1 );

moveNode( E, D, 2 );
